## 🔒 新的安全保证

### 代码级承诺：
1. ✅ **不读取兄弟 .env 文件** - `config.get_api_keys()` 只使用 `os.environ.get()`
2. ✅ **不操作路径** - 代码库中零个 `sys.path.insert()` 调用
3. ✅ **不运行时加载 .env** - 没有代码将 .env 文件解析到环境变量
4. ✅ **隔离的导入** - 所有模块只使用标准 Python 导入
5. ✅ **仅直接 API 调用** - 只访问 Twelve Data 和 Alpha Vantage 端点

### 代码验证：
```bash

$ grep -rn "sys.path.insert" *.py

# 未发现 load_dotenv（仅注释）
$ grep -rn "load_dotenv" *.py
1 个结果（安全注释） ✅

# 未发现 .env 文件打开
$ grep -rn "ENV_FILE\|\.env\.exists\|open.*\.env" *.py
```

## ⚠️ 破坏性变更

### 环境变量设置

**之前 (v1.3.1)**:
```bash
# 可以在项目目录中使用 .env 文件
echo "TWELVE_DATA_API_KEY=xxx" > .env
python3 cli.py
```

**之后 (v2.0.0)**:
```bash
# 必须通过外部环境变量设置
export TWELVE_DATA_API_KEY=xxx
export ALPHA_VANTAGE_API_KEY=yyy
python3 cli.py
```

### 依赖移除

**从 `requirements.txt` 和 `pyproject.toml` 中移除**:
- `python-dotenv>=1.0.0`（不再需要）
