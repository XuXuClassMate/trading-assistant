    💰 仓位管理¶

    版本: v1.5.0
    最后更新: 2026-03-25

    📊 概述¶

    仓位管理是风险控制的核心，帮助你合理分配资金，避免过度风险暴露。

    核心特性¶

    • ✅ 风险基础 - 基于可接受风险计算仓位
    • ✅ 多模式支持 - 固定金额/固定比例/风险基础
    • ✅ 自动计算 - 一键计算最优仓位
    • ✅ 止损建议 - 提供合理止损水平

    🎯 仓位计算方法¶

    1. 固定金额法¶

    原理: 每次交易固定金额

    公式: 

    仓位大小 = 固定金额 / 当前价格

    示例: - 固定金额：$1000 - BTC 价格：$67,500 - 仓位大小：0.0148 BTC

    优点: 简单直接
    缺点: 不考虑风险差异

    2. 固定比例法¶

    原理: 每次交易总资金的固定比例

    公式: 

    仓位金额 = 总资金 × 固定比例
仓位大小 = 仓位金额 / 当前价格

    示例: - 总资金：$10,000 - 固定比例：10% - BTC 价格：$67,500 - 仓位大小：0.0148 BTC

    优点: 资金增长时仓位自动增加
    缺点: 不考虑具体交易风险

    3. 风险基础法 (推荐)¶

    原理: 基于可接受损失计算仓位

    公式: 

    风险金额 = 总资金 × 风险比例
仓位大小 = 风险金额 / (入场价 - 止损价)

    示例: - 总资金：$10,000 - 风险比例：2% - 入场价：$67,500 - 止损价：$64,000 - 风险金额：$200 - 仓位大小：0.057 BTC ($3,857)

    优点: 风险可控，科学配比
    缺点: 计算稍复杂

    🛠️ 使用方法¶

    CLI 命令¶

    # 基本用法 (风险基础)
ta pos --symbol BTC --price 67500 --capital 10000 --risk 2

# 固定金额
ta pos --symbol BTC --price 67500 --amount 1000

# 固定比例
ta pos --symbol BTC --price 67500 --capital 10000 --percent 10

# 带止损
ta pos --symbol BTC --price 67500 --stop 64000 --capital 10000 --risk 2

    📊 实际应用¶

    仓位配置建议¶

    风险等级 单笔风险 最大仓位 适用场景
    保守 0.5-1% 10-20% 新手/不确定交易
    稳健 1-2% 20-40% 常规交易
    激进 2-3% 40-60% 高置信度交易
    专业 3-5% 60-100% 极高置信度

    止损设置¶

    常见方法:

    1. 百分比止损
    2. 保守：3-5%
    3. 稳健：5-8%

    4. 激进：8-12%

    5. 技术位止损

    6. 支撑位下方 2-3%
    7. 阻力位上方 2-3%

    8. ATR 的 2 倍

    9. 时间止损

    10. 设定持有时间
    11. 超时未达目标即离场

    ⚠️ 注意事项¶

    常见错误¶

    1. 仓位过大 - 单笔风险超过 5%
    2. 不止损 - 亏损时死扛
    3. 频繁交易 - 过度交易增加风险
    4. 情绪化 - 不按计划执行

    最佳实践¶

    1. 制定计划 - 交易前计算好仓位和止损
    2. 严格执行 - 按计划执行，不临时改变
    3. 记录总结 - 记录每笔交易，定期复盘
    4. 持续学习 - 不断改进仓位管理方法
